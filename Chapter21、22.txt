Chapter21.PL/SQL 外部プログラムからSTORED SUBPROGRAMを呼び出す
Chapter22.PL/SQL JAVA STORED PROCEDURE生成について

　외부 프로그램에서 호출한다는 의미는
　PL/SQL 에서만 서브프로그램을 호출하는것이아니라 다른 언어에서도 호출이 가능하다는 의미이다

　-------------------------------------------------------------------------------------------
　区分		説明
　-------------------------------------------------------------------------------------------
　입력		외부 프로그램에서 데이터베이스 서버로 값 전달
　			바인드변수가 서브프로그램의 IN 매개변수, 표현식의 r-value

　출력		데이터베이스 서버에서 외부 프로그램으로 값 전달
			바인드 변수가  OUT 매개변수, 표현식의 l-value

　입/출력	하나의 바인드변수만 사용하여 데이터베이스 서버로 값 전달하고, 실행 결과값을 다시 전달받는 유형
			IN OUT 매개변수, l-value, r-value 둘 다 사용가능

　반환		저장 함수가 반환
　			.NET 만 가능, JAVA 없음.
　-------------------------------------------------------------------------------------------
　
　21-1．JAVA에서 호출
　
　　-------------------------------------------------------------------------------------------
　　区分		説明
　　-------------------------------------------------------------------------------------------
　　입력		자바에서 데이터베이스로 값 전달.
				실행 전 값 입력.

　　출력		데이터베이스에서 자바로 값 전달
				실행전 받을 데이터 타입 지정해야함.

　　입/출력		입/출력 내용 혼합. 
				전달 전 값 입력, 반환 전 받을 값 지정
　　-------------------------------------------------------------------------------------------
　
　● 저장함수 호출
　　1) 익명 PL/SQL호출법
　　　BEGIN ? := factorial(?) ; END;
 
　　2) JDBC 을 이용한 저장 함수 호출
　　　{? =  call factorial(?) }
　　　첫번째 ? 는 출력
　　　두번째 ? 는 입력
　　　
　　※PlsalFunction.java　参照
　　※PlsqlFunctionUsingSelect.java　参照
　　
　● 저장프로시저 호출
　　저장 프로시저 예제
　　※compute_power.sql　参照
　　
　　1) 익명 PL/SQL 을 이용한 실행
　　　------------------------------------------------------------
　　　BEGIN
　　　  compute_power(?, ?, ?);
　　　END;
　　　------------------------------------------------------------
　　　
　　2) JDBC 를 이용한 호출
　　　------------------------------------------------------------
　　　{ call compute_power(?, ?, ?) }
　　　------------------------------------------------------------
